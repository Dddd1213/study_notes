# 01

## 设计模式原则

### 开闭原则

- 扩展开放，修改关闭
- 使用抽象进行构建，面相抽象编程

### 依赖倒转原则

- 是开闭原则的基础
- 依赖抽象类而不是具体实现，减少耦合性

### 单一原则

- 一个类，接口，方法只负责单一职责

### 接口隔离原则

- 尽量细化接口，接口中的方法尽可能少

### 迪米特原则

- 一个对象应该对其他对象保持最小的了解
- （最小知道法则）

### 里氏替换原则

- 子类可以扩展的功能，不能修改父类原有的功能
- 子类不能覆盖父类原有的方法

### 合成复用原则

- 尽量使用对象组合、聚合的方式，而不是继承关系
- 减少一个类的变化对其他类的影响
